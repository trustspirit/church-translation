# CLAUDE.md — 교회 번역 프로젝트

이 프로젝트는 예수 그리스도 후기 성도 교회 관련 영어 자료를 자연스럽고 정확한
한국어로 번역하는 작업이다. 번역 역할·원칙·문체·절차는 프로젝트 지침을 따른다.

## 번역 시 반드시 참고할 문서

- **[교회-번역-용어-참고집.md](교회-번역-용어-참고집.md)**
  공식 용어·조직명·출판물(1~4장), 문체·어체 원칙(5장), 인용·따옴표·영문 병기 규칙(6장),
  경전 인용 표기·처리 절차(7~8장), 지도자 말씀 인용 절차(9장), 맺음말 패턴(10장),
  번역 후 점검 체크리스트(11장)를 정리한 참고집.
  교회 용어·조직·문체·인용·맺음말 등이 등장하면 이 문서를 먼저 확인한다.
- **[교회-번역-인명-참고집.md](교회-번역-인명-참고집.md)**
  교회 지도자·경전 인물 이름, 직책·호칭 표기 규칙을 정리한 참고집.
  인명이 등장하면 이 문서를 먼저 확인한다.

## 핵심 원칙 요약

- 교회의 공식 한국어 용어를 우선한다. 불확실하면 임의로 만들지 말고 `[확인 필요]` 표시.
- 인명: 공식 한국어 표기 우선. 교회 지도자 이름은 참고집의 표기 규칙을 따른다.
- 참고집의 표기도 의심스러우면 churchofjesuschrist.org 한국어판으로
  재검증한다. 참고집은 anchor 정의 모음이지 절대 권위가 아니다.
- 경전·지도자 말씀 직접 인용은 미리 정리해 두지 않는다. 인용 시
  churchofjesuschrist.org 한국어판에서 공식 문구를 웹 검색으로 찾아
  그대로 사용한다. (용어 참고집 8·9장 절차 참조)
- 원문에 없는 해석·설명·방어적 표현을 추가하지 않는다.
- **한국어 문체·어체** (상세는 용어 참고집 5장):
  - 합쇼체로 통일 (`-습니다`, `-드립니다`, `-십시오`), 청중은 "여러분"
  - 하나님·구주의 영어 인칭대명사(He/Him/His)는 "그"로 직역하지 말고
    "주님 / 하나님 / 그분" 등 호칭으로 옮긴다
  - 영어 직역(수동태·긴 관계절·추상명사 나열·강조어 직역)을 피하고
    한국어 호흡에 맞게 능동·동사구·짧은 문장으로 재구성한다

## 우선순위 순서

충돌이 있을 때 다음 순서를 따른다:

1. **공식 한국어 경전** (churchofjesuschrist.org 한국어 경전)
2. **총회 대회·교회 출판물 공식 한국어 번역**
3. **참고집** (용어 참고집, 인명 참고집)
4. **용어 참고집 5장 문체 원칙에 따른 자연스러운 한국어**

## 경전 인용 규칙

### 직접 인용 → 경전체 그대로

경전을 큰따옴표 안에 직접 인용할 때는 churchofjesuschrist.org 한국어 경전 문구를
한 글자도 바꾸지 않고 그대로 쓴다.

| 교리와 성약 6:36 공식 한국어 | "무슨 생각을 하든지 나를 바라보라. 의심하지 말며, 무서워 말라." |
|---|---|

- "두려워하지 말라" ≠ 공식 경전 문구. 공식 한국어는 **무서워 말라**.
- 동의어로 대체하거나 경전체를 합쇼체로 바꾸지 않는다.

### 화자가 경전 내용으로 권면할 때 → 합쇼체, 단 공식 경전 어휘 사용

**두 층위 규칙:**
- **어휘**: 공식 한국어 경전 표현을 따른다 (예: "두려워하다" 대신 "무서워하다")
- **형식**: 직접 인용(큰따옴표)이 아니면 합쇼체를 사용한다

경전체(말라, 바라보라)는 큰따옴표 안의 경전 구절에만 쓴다. 화자가 청중에게
자신의 목소리로 같은 내용을 전할 때는, 경전 구절을 그대로 반영하더라도 합쇼체를 쓴다.

**교리와 성약 6:36 예시:**
```
경전 직접 인용 (큰따옴표 안):
  "무슨 생각을 하든지 나를 바라보라. 의심하지 말며, 무서워 말라."
  → 경전체 그대로 유지 ✓

화자가 청중에게 권면할 때 (직접 인용 아님):
  "의심하지 말며, 무서워하지 마십시오."    ← 어휘: 무서워 ✓  형식: 합쇼체 ✓
  "무슨 생각을 하든지 그분을 바라보십시오." ← 어휘: adapted   형식: 합쇼체 ✓
```

### 혼잣말·내적 독백 인용 → 해체/반말

지도자가 어린 시절 혼잣말이나 내적 독백을 인용할 때는 해체/반말을 사용한다.
합쇼체 말씀 안에서도 의도적인 문체 전환이며 오류가 아니다.

**예시:** 넬슨 회장 2018년 총회 대회 공식 한국어 번역:
```
"자라면서 ... 저는 혼잣말로 이렇게 말하곤 했습니다.
 '이제 크리스마스 선물은 필요없어! 내가 바라는 선물은 내 부모님께 인봉되는 거야.'"
```
서술부는 합쇼체("말하곤 했습니다"), 작은따옴표 안 혼잣말은 해체("필요없어!", "거야").

| 규칙 | 적용 |

| 본문 전체 | 합쇼체 (-습니다, -십시오) |
| 청중 호칭 | "여러분" |

| 미확인 용어 | `[확인 필요]` |

## 번역 후 Review 절차 (필수)

번역 후 자체 review를 수행하고 `[확인 필요]`로 표시된 항목은 그대로 두지
말고 가능한 한 웹 검색으로 검증하여 교정한다.

1. `[확인 필요]` 항목(공식 용어, 인명, 조직명, 지도자 말씀 인용, 경전
   문구, 말씀 제목 등)을 모두 모은다.
2. 각 항목을 웹 검색으로 검증한다. 1차 출처는 churchofjesuschrist.org
   한국어판과 교회 뉴스룸 한국어판이다.
3. 공식 한국어 표기·문구를 찾으면 번역문을 그에 맞게 교정한다.
   - 지도자 말씀 직접 인용은 공식 한국어 번역문을 그대로 사용한다.
   - 인용 출처(말씀 제목·모임명)도 공식 한국어 명칭으로 바꾼다.
4. 검증으로 확정한 용어·조직·경전 문구는 교회-번역-용어-참고집.md에,
   인명은 교회-번역-인명-참고집.md에 추가하여 다음 번역에서 재사용한다.
5. 웹 검색으로도 확인할 수 없는 항목만 최종적으로 `[확인 필요]`로 남기고,
   검증을 시도했으나 확인하지 못했음을 밝힌다.

## 링크 번역 절차

사용자가 URL을 주면 유형을 판단해 원문을 추출한 뒤 번역한다.
**번역·리뷰·출력 규칙은 영어 원문을 직접 붙여넣었을 때와 동일하다.**
이 절차는 원문 확보 단계만 추가하는 것이다.

```
URL 제공?
├── YouTube (youtube.com / youtu.be)
│   └── scripts/extract-youtube.sh <URL> 실행
│       .cache/<video-id>/meta.json + transcript.txt 읽기
│       → transcript.txt를 원문으로 사용
└── 일반 웹 링크
    └── WebFetch로 페이지 가져오기
        본문(말씀·기사·교과) 추출, 내비·광고·푸터 제거
        → 추출한 본문을 원문으로 사용
```

### YouTube 링크

1. `scripts/extract-youtube.sh <URL>` 실행.
   - 공식 자막 → 자동 자막 → Whisper STT 순으로 영어 스크립트를 만든다.
   - 의존 도구가 없으면 스크립트가 설치 명령을 안내한다.
2. `.cache/<video-id>/meta.json`(title, uploader, description, upload_date)과
   `transcript.txt` 읽기.
3. 메타데이터 보정:
   - **제목**: `meta.title` 기반, 채널명·부제 등 군더더기 정리.
   - **말씀하신 분**: `uploader`는 보통 채널명이므로 신뢰하지 않는다.
     제목·`description`·본문에서 실제 화자를 추론하고, 불확실하면 `[확인 필요]`.
   - **날짜**: `upload_date`는 실제 말씀 날짜와 다를 수 있다.
     `description`·본문 단서로 보정하고, 불확실하면 `[확인 필요]`.
4. 자막·STT가 모두 실패해 스크립트가 비0으로 종료해도 `meta.json`은 남는다.
   이 경우 메타데이터만 사용자에게 보고하고 원문을 직접 받을 수 있는지 묻는다.

### 일반 웹 링크

1. `WebFetch`로 페이지 가져오기.
2. 말씀·기사·교과 본문만 추출. 내비, 사이드바, 푸터, 광고는 제거.
3. 페이지에서 제목·화자·날짜 확인. 불확실하면 `[확인 필요]`.
4. 로그인 필요 또는 접근 불가 시 사용자에게 보고하고 원문을 직접 받는다.

## 번역 결과물 출력 규칙

- **번역 최종본은 `translations/` 폴더에 별도 md 파일로 저장한다.**
  - 파일명: 원문 제목을 기반으로 한 한국어 슬러그. 예: `translations/교회-인공-지능-사용-원리-발표.md`
  - 폴더가 없으면 만든다.
  - 파일에는 번역 본문만 담는다. `[확인 필요]` 표시는 본문 안에 그대로 유지한다.
  - 원문이 함께 필요하면 파일 맨 아래 `---` 구분선 뒤에 "원문" 섹션으로 덧붙여도 된다 (요청 시).
- **번역 본문은 md 파일에만 저장하고 채팅에 다시 옮기지 않는다.**
  사용자는 파일을 직접 열어서 확인한다. 채팅에 번역 본문을 통째로 붙여 넣지 않는다.
- **리뷰 및 확인 사항(검증 완료 항목, `[확인 필요]`로 남긴 항목, 출처 링크,
  통찰 등)만 채팅에 표시한다.**
  - 후속 수정 요청이 오면 파일을 수정하고, 수정 내역과 새 `[확인 필요]` 항목을
    채팅에 간단히 정리한다.

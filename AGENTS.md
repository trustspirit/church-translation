# 교회 번역 프로젝트 공통 지침

이 프로젝트는 예수 그리스도 후기 성도 교회 관련 영어 자료를 자연스럽고 정확한
한국어로 번역하는 작업이다. 번역 역할·원칙·문체·절차는 프로젝트 지침을 따른다.

## 지침 파일 동기화 규칙 (필수)

- `AGENTS.md`와 `CLAUDE.md`는 항상 바이트 단위로 동일한 내용을 유지한다.
- 둘 중 한 파일을 추가·수정·삭제하면 같은 작업 안에서 다른 파일에도 동일한
  변경을 즉시 반영한다. 어느 한쪽만 변경된 상태로 작업을 끝내지 않는다.
- 두 파일 중 어느 쪽에서 시작한 변경이든 다른 쪽으로 동기화한 뒤,
  `cmp -s AGENTS.md CLAUDE.md`로 완전 일치를 확인해야 작업이 완료된다.

## 번역 시 반드시 참고할 문서

- **[교회-번역-용어-참고집.md](교회-번역-용어-참고집.md)**
  공식 용어·조직명·출판물(1~4장), 문체·어체 원칙(5장), 인용·따옴표·영문 병기 규칙(6장),
  경전 인용 표기·처리 절차(7~8장), 지도자 말씀 인용 절차(9장), 맺음말 패턴(10장),
  번역 후 점검 체크리스트(11장)를 정리한 참고집.
  교회 용어·조직·문체·인용·맺음말 등이 등장하면 이 문서를 먼저 확인한다.
- **[교회-번역-인명-참고집.md](교회-번역-인명-참고집.md)**
  교회 지도자·경전 인물 이름, 직책·호칭 표기 규칙을 정리한 참고집.
  인명이 등장하면 이 문서를 먼저 확인한다.

## 핵심 원칙 요약

- 교회의 공식 한국어 용어를 우선한다. 불확실하면 임의로 만들지 말고 `[확인 필요]` 표시.
- 인명: 공식 한국어 표기 우선. 교회 지도자 이름은 참고집의 표기 규칙을 따른다.
- 참고집의 표기도 의심스러우면 churchofjesuschrist.org 한국어판으로
  재검증한다. 참고집은 anchor 정의 모음이지 절대 권위가 아니다.
- 경전·지도자 말씀 직접 인용은 미리 정리해 두지 않는다. 인용 시
  churchofjesuschrist.org 한국어판에서 공식 문구를 웹 검색으로 찾아
  그대로 사용한다. (용어 참고집 8·9장 절차 참조)
- 원문에 없는 해석·설명·방어적 표현을 추가하지 않는다.
- **한국어 문체·어체** (상세는 용어 참고집 5장):
  - 합쇼체로 통일 (`-습니다`, `-드립니다`, `-십시오`), 청중은 "여러분"
  - 하나님·구주의 영어 인칭대명사(He/Him/His)는 "그"로 직역하지 말고
    "주님 / 하나님 / 그분" 등 호칭으로 옮긴다
  - 영어 직역(수동태·긴 관계절·추상명사 나열·강조어 직역)을 피하고
    한국어 호흡에 맞게 능동·동사구·짧은 문장으로 재구성한다

## 우선순위 순서

충돌이 있을 때 다음 순서를 따른다:

1. **공식 한국어 경전** (churchofjesuschrist.org 한국어 경전)
2. **총회 대회·교회 출판물 공식 한국어 번역**
3. **참고집** (용어 참고집, 인명 참고집)
4. **용어 참고집 5장 문체 원칙에 따른 자연스러운 한국어**

## 경전 인용 규칙

### 직접 인용 → 경전체 그대로

경전을 큰따옴표 안에 직접 인용할 때는 churchofjesuschrist.org 한국어 경전 문구를
한 글자도 바꾸지 않고 그대로 쓴다.

| 교리와 성약 6:36 공식 한국어 | "무슨 생각을 하든지 나를 바라보라. 의심하지 말며, 무서워 말라." |
|---|---|

- "두려워하지 말라" ≠ 공식 경전 문구. 공식 한국어는 **무서워 말라**.
- 동의어로 대체하거나 경전체를 합쇼체로 바꾸지 않는다.

### 화자가 경전 내용으로 권면할 때 → 합쇼체, 단 공식 경전 어휘 사용

**두 층위 규칙:**
- **어휘**: 공식 한국어 경전 표현을 따른다 (예: "두려워하다" 대신 "무서워하다")
- **형식**: 직접 인용(큰따옴표)이 아니면 합쇼체를 사용한다

경전체(말라, 바라보라)는 큰따옴표 안의 경전 구절에만 쓴다. 화자가 청중에게
자신의 목소리로 같은 내용을 전할 때는, 경전 구절을 그대로 반영하더라도 합쇼체를 쓴다.

**교리와 성약 6:36 예시:**
```
경전 직접 인용 (큰따옴표 안):
  "무슨 생각을 하든지 나를 바라보라. 의심하지 말며, 무서워 말라."
  → 경전체 그대로 유지 ✓

화자가 청중에게 권면할 때 (직접 인용 아님):
  "의심하지 말며, 무서워하지 마십시오."    ← 어휘: 무서워 ✓  형식: 합쇼체 ✓
  "무슨 생각을 하든지 그분을 바라보십시오." ← 어휘: adapted   형식: 합쇼체 ✓
```

### 혼잣말·내적 독백 인용 → 해체/반말

지도자가 어린 시절 혼잣말이나 내적 독백을 인용할 때는 해체/반말을 사용한다.
합쇼체 말씀 안에서도 의도적인 문체 전환이며 오류가 아니다.

**예시:** 넬슨 회장 2018년 총회 대회 공식 한국어 번역:
```
"자라면서 ... 저는 혼잣말로 이렇게 말하곤 했습니다.
 '이제 크리스마스 선물은 필요없어! 내가 바라는 선물은 내 부모님께 인봉되는 거야.'"
```
서술부는 합쇼체("말하곤 했습니다"), 작은따옴표 안 혼잣말은 해체("필요없어!", "거야").

## 번역 후 Review 절차 (필수)

번역 후 자체 review를 수행하고 `[확인 필요]`로 표시된 항목은 그대로 두지
말고 가능한 한 웹 검색으로 검증하여 교정한다.

1. `[확인 필요]` 항목(공식 용어, 인명, 조직명, 지도자 말씀 인용, 경전
   문구, 말씀 제목 등)을 모두 모은다.
2. 각 항목을 웹 검색으로 검증한다. 1차 출처는 churchofjesuschrist.org
   한국어판과 교회 뉴스룸 한국어판이다.
3. 공식 한국어 표기·문구를 찾으면 번역문을 그에 맞게 교정한다.
   - 지도자 말씀 직접 인용은 공식 한국어 번역문을 그대로 사용한다.
   - 인용 출처(말씀 제목·모임명)도 공식 한국어 명칭으로 바꾼다.
4. 검증으로 확정한 용어·조직·경전 문구는 교회-번역-용어-참고집.md에,
   인명은 교회-번역-인명-참고집.md에 추가하여 다음 번역에서 재사용한다.
5. 웹 검색으로도 확인할 수 없는 항목만 최종적으로 `[확인 필요]`로 남기고,
   검증을 시도했으나 확인하지 못했음을 밝힌다.

## 자기 개선 절차 (번역 종료 시 필수)

번역·리뷰·출력을 마친 뒤, 이번 작업에서 새로 배운 것을 다음 번역에서
재사용할 수 있도록 기록한다. 무엇을 어디에 적을지는 유형으로 나눈다.

| 새로 얻은 것 | 기록 위치 |
|---|---|
| 확정된 공식 용어·조직명·출판물·경전/말씀 문구 | 교회-번역-용어-참고집.md |
| 확정된 인명·직책 표기 | 교회-번역-인명-참고집.md |
| 처리 방식·절차·노하우, 반복되는 실수 패턴 | 이 문서의 "처리 노하우 누적" |

절차:

1. 이번 번역에서 효과적이었던 처리 방식, 막혔던 지점과 해결법, 새로 깨달은
   함정을 한두 줄로 정리한다.
2. 용어·인명·인용이면 해당 참고집에 추가한다(위 Review 절차 4단계와 동일).
3. 절차·노하우면 아래 "처리 노하우 누적"에 항목을 더한다. 이미 있는 항목과
   겹치면 합치고, 틀린 항목은 고친다. 일회성 사실은 적지 않는다.
4. 여러 번 반복돼 일반 원칙으로 굳은 노하우는 본문 해당 섹션(문체 빠른 참고,
   흔한 실수 표 등)으로 승격한다.
5. 무엇을 어디에 추가·수정했는지 채팅에 한 줄로 보고한다.

### 처리 노하우 누적

- **인용이 많은 말씀은 초안 전에 공식 문구부터 확보한다.** 경전·지도자 인용이
  여러 곳이면 본문을 의역으로 채우기 전에 공식 한국어 문구를 먼저 모은다.
  병렬 서브에이전트로 (1) 신약·구약, (2) 몰몬경, (3) 교리와 성약·값진 진주,
  (4) 지도자 인용으로 묶어 동시 조회하면 정확도·속도가 함께 오른다. 각
  에이전트에는 "직접 번역하지 말고 공식 한국어 문구만 verbatim 보고" 지시.
- **인용 출처는 화자의 귀속을 그대로 믿지 말고 인용별로 따로 검증한다.** 화자가
  여러 인용을 한 출처(예: 베드나 "Seek Learning by Faith")로 묶어 제시해도, 실제로는
  각기 다른 말씀·인터뷰에서 온 경우가 있다. 인용문 한 줄씩 검색해 진짜 출처를
  찾는다. 일부는 교회 뉴스 인터뷰 등 **영어 전용 자료**라 공식 한국어가 없을 수
  있으니 그때만 임시 번역 + `[확인 필요]`. (Gilbert 2024 졸업식 사례: "행동과 힘의
  원리"는 "믿음으로 구하고"(2008), "주의 종으로서…"는 2021년 교회 뉴스 인터뷰,
  "배우는 자로서…"만 "신앙으로 배움을 추구함"에서 옴.)
- **인용문을 공식 한국어 출처에서 찾으면 문구뿐 아니라 날짜와 출전도 대조한다.**
  원문 말씀의 연도·쪽수 표기가 공식 교재 각주와 충돌하면 공식 1차 출처에 맞춰
  교정하고, 번역 리뷰에서 원문 오류와 교정 내용을 밝힌다. (매드슨 “올리브
  압착기”의 조셉 스미스 예언: 원문 1842년 → 공식 출처 1843년 사례.)
- **직책 음역은 참고집 기존 항목도 churchofjesuschrist.org/news-kr로 재검증한다.**
  Commissioner of CES는 "위원장"도 "교육감"도 아닌 **"교회 교육 기구 총감"**.
  인물 소개 페이지(`/learn/{name}?lang=kor`)나 교회 뉴스룸 한국어판에서 확인한다.
- **공식 한국어가 없는 인용 처리.**
  - 탈매지 『예수 그리스도』 등 일부 책은 복음 자료실에 영어만 있다. 한국어
    교재 인용을 검색해 보고, 없으면 임시 번역 + `[확인 필요 — 공식 한국어 문구 미확인]`.
  - 1970년대 등 오래된 연차 대회 말씀은 한국어 본문이 온라인에 없을 수 있다.
    인용 속 경전 구절만 공식 경전체로 처리하고, 화자 고유 표현은 임시 번역 +
    `[확인 필요]`. 단 『교회 회장들의 가르침』 같은 교재에 같은 일화의 다른
    발췌가 있으면 그 공식 문구를 활용한다.
- **19세기 1차 사료 인용도 최신 교재의 재인용을 먼저 찾는다.** 원 일지·잡지에
  한국어판이 없어도 최근 세미나리 교재나 교회 역사 자료가 같은 일화와 문장을
  공식 한국어로 발췌했을 수 있다. 인물명과 독특한 문구를 함께 검색해 찾은
  부분만 verbatim으로 사용한다(지나 디 에이치 영의 몰몬경 일화 사례).
- **찬송가 공식 한국어 가사가 영문을 크게 의역했으면 혼합 인용을 만들지 않는다.**
  큰따옴표 안에는 공식 한국어 가사만 그대로 쓰고, 영문에만 있는 개념은 화자의
  연결 문장을 자연스럽게 재구성해 살린다("O My Father" 2절 사례).
- **본사이트 `?lang=kor`가 404여도 성도의 벗 공식 번역이 한국 아카이브에 있을 수 있다.**
  1990~2000년대 연차 대회 말씀은 churchofjesuschrist.org 본사이트 kor URL이 404여도,
  **kr.churchofjesuschrist.org** 한국 자료실에 『성도의 벗』 번역 전문이 남아 있는
  경우가 있다. 경로 패턴:
  `kr.churchofjesuschrist.org/church-materials/korea-resources/church-magazine/conference-report/liahona/<YYYY>/<MM>/<영문-슬러그>?lang=kor-kr`
  (총회 다음 분기 성도의 벗 호에 실림 — 1998.10 대회 → 1999.01호). 이 페이지는
  서버 렌더링이라 `curl -sL "<url>" -H "User-Agent: Mozilla/5.0"`로 본문 전문을
  받을 수 있다. **말씀 전문에 공식 번역이 있으면 그것이 우선순위 2**이므로,
  새로 번역하지 말고 이를 기반으로 삼는다(사용자에게 재번역 vs 공식본 정리 확인).
- **옛 성도의 벗 공식본을 "현행표기 정리"할 때의 규칙.** 1990년대 번역은 옛 표기·
  오탈자를 쓴다: **대관장→회장**, **대관장단→제일회장단**, **스테이크 부장→스테이크 회장**,
  **구세주→구주**(단 직접 인용 안에서는 공식이 아직 구세주를 쓰면 유지), **요셉 스미스→조셉 스미스**,
  **더블류→더블유**, 미디아→미디어 등. 신성 호칭 `께서`·`간증합니다` 등 문체 규칙도
  현행화. **그러나 직접 인용은 원칙이 다르다**: 경전 직접 인용은 옛 성도의 벗 인용을
  버리고 **현행 공식 경전**(우선순위 1)으로 교체한다(예: 교리와 성약 128:22·24는 옛
  번역이 아니라 현행 경전 문구로). 지도자·교재 인용은 그 인용의 현행 공식 출처가
  있으면 거기에 맞추고, 없으면 옛 성도의 벗 문구를 유지하되 이름·용어만 현행화한다.
- **최근·미래 날짜 행사는 한국어 페이지가 아직 없을 수 있다.** broadcasts 등의
  `?lang=kor`가 404면 행사명·말씀 제목을 임시 번역 + `[확인 필요]`로 두고
  영어판 본문으로 번역한다.
- **맺음말 동사는 본문의 마지막 행위로 결정한다.** 마지막이 사도의 축복
  ("I bless you … In the sacred name of …")이면 "예수 그리스도의 성스러운
  이름으로 축복합니다. 아멘."(용어 참고집 10장 표와 연결).
- **말씀 텍스트 페이지도 전사 누락·오류가 있을 수 있다.** 문장이 뜬금없이
  끊기거나(예: "Isaiah said" 뒤에 대문자로 새 문장 시작), 화자가 했을 리 없는
  말이 귀속돼 있으면 전사 오류를 의심한다. 같은 말씀의 **YouTube 영상 자막**을
  `scripts/extract-youtube.sh`로 받아 대조하면 누락 문장을 복원할 수 있다.
  (BYUI speeches 페이지에서 "it simply as, 'Do not be dismayed.' Please hear
  that warning voice." 한 구절이 통째로 빠져 "Isaiah said"만 남았던 사례.)
- **말씀 페이지의 경전 링크·장절도 오기일 수 있다.** 링크가 있어도 그대로
  믿지 말고 해당 책의 장 수와 인용 문맥을 공식 경전에서 대조한다. 특히 한 장뿐인
  책은 장·절이 잘못 조합되기 쉽다(BYU Speeches의 `Enos 7:8` → 실제로는
  **이노스서 1:7~8**이었던 사례).
- **화자가 음원에서도 경전에 없는 문장을 성구처럼 덧붙이면 두 층위로 분리한다.**
  실제 발언은 보존하되, 공식 경전과 일치하는 부분만 큰따옴표 안에 verbatim으로
  두고 성구에 없는 삽입은 따옴표 밖에서 화자의 해설로 옮긴다. 이렇게 하면 화자의
  뜻을 빼지 않으면서도 비공식 문구가 경전체로 오인되는 일을 막을 수 있다
  (헤일즈 1978년 "Principles of Sacrifice"의 창세기 22:16~18 인용 사례).
- **BYU Speeches에 본문이 없다고 표시되어도 음원과 정식 재게재 원고를 함께 찾는다.**
  페이지 HTML의 `AudioObject` 또는 `contentUrl`에서 mp3를 받고, 말씀 제목을 검색해
  저자 선집·교회 계열 잡지 등에 실린 편집 원고를 찾는다. 편집 원고를 본문 뼈대로
  쓰되 음원 전사와 대조하여 도입·구두 예화·맺음말의 누락을 보완한다. 두 자료가
  서로 다른 대목은 음원과 후대 공식 역사 자료로 재검증한다.
- **BYU·종교 교육원 영적 모임(devotional·speeches)은 본문이 다른 사이트로
  연결된다.** `byui.edu/speeches/...`, `speeches.byu.edu/...` 페이지는 제목·
  화자·날짜만 있고 본문은 "Visit ... to read"로 churchofjesuschrist.org
  교재(`/study/manual/...`)에 연결된다. 그 URL을 WebFetch해 본문을 확보한다.
- **이런 영적 모임 말씀은 공식 한국어 번역이 없는 경우가 많다.** 제목은 직접
  옮기되, 제목이 경전 문구이면(예: "That Ye May Believe" → 요한복음 20:31
  "너희로 ... 믿게 하려 함") 공식 경전 표현을 살려 번역하고, 공식 한국어
  제목이 아님을 채팅에 밝힌다.
- **speeches.byu.edu 페이지 본문은 curl로 안 보인다 — WordPress REST API로
  받는다.** 이 사이트는 SPA라 페이지 HTML에 본문이 없다(`single-speech__content`
  클래스만 있고 실제 텍스트는 빈 채). 대신 `<link rel="alternate" ... wp-json/wp/v2/speech/<ID>>`
  주소가 head에 있으므로 `curl -sL "https://speeches.byu.edu/wp-json/wp/v2/speech/<ID>"`로
  JSON을 받아 `content.rendered` 필드를 HTML 태그 제거 후 쓰면 된다. 이 편집
  게재본은 YouTube 자동 자막보다 훨씬 정확하고(구두체 오류·잘못 들은 단어 없음),
  각주(성구·서적 인용)까지 포함돼 있어 YouTube 링크로 받은 말씀이라도 설명란에
  BYU Speeches 링크가 있으면 그쪽을 원문으로 우선한다.
- **BYU Speeches가 오디오만 제공하는 경우에는 JSON-LD를 먼저 확인한다.**
  REST API의 `content.rendered`가 비어 있고 페이지가 "The text for this speech
  is unavailable"이라고 명시하면, HTML의 JSON-LD `audio.contentUrl`에서 MP3를
  받아 전사한다. `mentions`·`teaches`에는 인명·성구·주제 단서가 있어 난청
  구간과 인용 출처를 찾는 데 유용하다. 오래된 오디오는 먼저 Whisper small로
  전체 타임스탬프를 만든 뒤, 불분명한 구간만 medium/large-v3로 다시 전사하면
  정확도와 시간을 함께 확보할 수 있다. (매드슨 1965년 BYU 영적 모임 사례.)
- **데이비드 오 맥케이 말씀은 BYU Speeches에 음원만 있어도 맥케이 교육대학원
  아카이브를 함께 검색한다.** `education.byu.edu/mckay/<연월일>` 형식의
  페이지에 같은 음원의 편집 게재 원고가 남아 있을 수 있다. 음원 전사보다 그
  원고를 본문 뼈대로 우선하되, 인명·지명·단어의 명백한 오기는 음원과 공식
  교회사로 교차 확인한다. 두 페이지의 날짜가 다르면 사용자가 준 BYU Speeches
  날짜를 결과물에 쓰고 리뷰에서 불일치를 밝힌다(맥케이 1952년 “Message for
  LDS College Youth” 사례: 10월 10일 vs 게재 원고 10월 8일).
- **BYU Speeches MP3 원본 호스트가 지나치게 느리면 CloudFront 경로를 확인한다.**
  JSON-LD의 `audio.contentUrl`이 `speeches.byu.edu/wp-content/...`인데 다운로드가
  멈추면, 같은 `/wp-content/...` 경로를
  `https://d6zb2yxvzmqfc.cloudfront.net`에 붙여 요청한다. 먼저 HEAD의 파일 크기와
  ETag를 대조한 뒤 동일 파일일 때만 전사에 사용한다.
- **오래된 BYU 영적 모임 말씀에서 "House of Glory" 류의 표현이 나오면 D&C
  88:119 / 109:8을 확인한다.** 두 구절은 문구가 동일하며 "영광의 집" 등 성전의
  일곱 가지 명칭을 담고 있어, 관련 말씀 제목·본문 표현의 공식 번역 근거가 된다.
- **단, BYU 영적 모임이 공식 한국어 『리아호나』 기사로 출판됐을 수 있다.**
  말씀 제목으로 `study/liahona` 한국어판을 검색한다. 그 기사는 전문을 줄인
  **축약본**일 때가 있어(예화·도표 통째 생략) 본문 번역을 대체하지 못한다.
  그러나 그 안의 경전·지도자 인용은 이 말씀에 대한 공식 번역이므로 인용문
  표현을 거기에 맞춘다(우선순위 2 > 내 번역). 빠진 본문은 영어 전문에서 옮긴다.
  (패커 2007년 BYU 영적 모임 "리하이의 꿈과 여러분" → 『리아호나』 2010년
  8월호 "리하이의 꿈에 우리 모습이 나옵니다" 사례: 도입부·종교 교육원 노래·
  조셉 에프 스미스 일화·맺음말은 빠지고 핵심 본문은 공식 번역됨.)
- **BYU 말씀 전문의 공식 한국어판이 없어도 후대 『교회 회장들의 가르침』에
  여러 단락이 재수록됐을 수 있다.** 화자가 교회 회장이면 해당 회장 교재의
  주제별 장과 각주에서 영어 말씀 제목을 검색한다. 원문과 정확히 겹치는 단락은
  교재의 공식 한국어를 verbatim으로 쓰고, 생략되거나 재구성된 부분만 영어
  전문에서 새로 옮긴다(힝클리 1997년 "These Noble Pioneers" 사례).
- **한 인용문의 출처가 교재와 공식 기사에 나뉘어 있을 때.** 인용의 일부는
  교재 발췌에 없고(생략 표시 …) 공식 기사에만 있을 수 있다. 양쪽 공식 문구를
  발췌해 합치되, 채팅 리뷰에 어느 문장이 어느 출처인지 분리해 밝힌다.
  BYU 말씀의 영어 제목을 교회 사이트에서 검색하면 공식 한국어 전문이 없어도
  성전 주제 페이지나 한국어 학생 교재 PDF에 일부 단락이 번역되어 있을 수 있다.
  제목이 영어로 남아 있어도 본문 인용은 공식 한국어일 수 있으므로 함께 확인한다.
- **경전/교재는 JS 렌더링이라 WebFetch가 요약·오탈자를 낼 수 있다.** 직접
  인용은 정확도가 생명이므로, 미심쩍으면 playwright로 `navigate` 후 `.body-block`
  innerText를 추출해 verbatim 확보한다(말씀 전문도 이 방법이 안전). WebFetch가
  엉뚱한 절을 반환하는 일도 있으니(예: 니파이후서 25:23 대신 25:21) 핵심 구절은
  절 번호로 교차 확인한다.
- **브라우저 확장이 없을 때는 curl + python으로도 verbatim 확보가 된다.**
  churchofjesuschrist.org 경전/총회 말씀 페이지는 실제로는 서버 렌더링이라
  `curl -sL "<url>?lang=kor" -H "User-Agent: Mozilla/5.0"` 로 HTML을 받으면
  본문이 들어 있다. 경전 절은 `<p id="pN">`, 말씀 단락은 `class="body-block"`
  으로 표시되므로 python(re)로 해당 태그만 뽑아 `<sup>`(각주 링크) 제거 후
  절 번호로 교차 확인한다. WebFetch(작은 모델 요약)보다 이쪽이 direct 인용에 안전.
- **인용이 많은 학술 장(RSC 등)은 병렬 서브에이전트 3분할이 효율적이다.**
  (1) 교리와 성약, (2) 성경·몰몬경, (3) 지도자 인용으로 나눠 verbatim만 보고받고,
  맥스웰 등 direct 인용은 메인에서 위 curl 방식으로 최종 재확인한다.
- **"live the law" 직역 "살아 내다" 회피.** 영어 "live the law / be able to live
  these laws"를 "법을 살아 내다"로 옮기면 어색하고, 여러 번 나오면 특히 거슬린다.
  문맥에 따라 "법대로 살다 / 지켜 행하다 / 실천하다"로 변주한다. 또 "celestial ~"는
  바로 인용된 교리와 성약 88:22와 맞춰 통일한다: celestial law=해의 왕국의 율법,
  celestial salvation=해의 왕국의 구원, celestial reward=해의 영광("해의 영광에 이르다";
  "해의 상급"은 어색), celestial glory=해의 영광(88:22 경전체 그대로). (맥콩키 1975
  "순종, 헌납, 희생" 사례. 이 말씀은 온라인 공식 한국어 전문이 없어 다듬어 번역.)
- **consecration 용어 이중성.** 재산·자신을 바치는 "law of consecration"은
  **헌납의 법**(성별의 법 아님). 단 2니파이 32:9 "행함을 성별", D&C 42:39
  "성별할 것"처럼 공식 경전이 특정 절에서 "성별"을 쓰면 그 절은 verbatim 유지.
- **각주로 인용 범위를 확정한다.** 본문 인용에 "…"이 있으면 영어 각주의 절
  범위(예: "Helaman 16:6, 13–18, 20")로 어느 절을 빼는지 정확히 맞춘다.
- **공식 한국어 교재가 HTML이 아니라 PDF로만 있을 때.** 『교회 회장들의 가르침』
  계열은 `…/language-materials/<품번>_kor.pdf`로 받아 pypdf(python)로 텍스트
  추출 후 grep해 단락을 verbatim 확보한다(예: TPC: Joseph Smith = 36481_kor.pdf).
  경로는 `english/pdf`가 아니라 **`korean/pdf`**여야 한다: 정확한 전체 URL은
  `https://www.churchofjesuschrist.org/bc/content/shared/content/korean/pdf/language-materials/36481_kor.pdf?lang=kor`.
- **TPJS/WJS 인용이 TPC:JS(교재)에 없으면 리아호나 기사도 검색한다.** 유명한
  조셉 스미스 어록(예: "하나님의 성품을 확실히 아는 것이 복음의 첫째 원리")은
  TPC:JS 해당 장에 없어도 리아호나 기사(예: 2016년 1월호 "신회를 이해함")에
  같은 문장이 공식 번역으로 인용돼 있을 수 있다. 한 인용의 앞·뒤 절이 서로
  다른 출처(교재/기사)에서만 각각 확인되는 경우, 확인된 부분만 verbatim으로
  쓰고 나머지는 `[확인 필요]`로 남긴다.
- **TPJS와 TPC:JS는 다른 책이다.** 각주의 "Teachings: Joseph Smith"는
  『교회 회장들의 가르침: 조셉 스미스』(TPC:JS, HTML/PDF 있음), "Teachings of
  the Prophet Joseph Smith"는 『선지자 조셉 스미스의 가르침』(TPJS, 별책)이다.
  TPJS는 churchofjesuschrist.org에 한국어가 없을 수 있어 그때는 임시 번역 +
  `[공식 번역 확인 필요]`로 둔다.
  - **TPJS 인용이 많은 글은 TPC:JS PDF 일괄 대조가 가치 있다.** TPC:JS는
    TPJS를 주제별로 재구성·발췌한 교재라 TPJS 인용문 중 상당수(다만 전부는
    아님)가 겹친다. 인용이 10개 이상이면 서브에이전트에 TPC:JS PDF
    (`36481_kor.pdf`)를 내려받아 각 인용을 주제·의미로(문자열 그대로가
    아니라) 검색시키고, 찾은 것만 verbatim 보고받는다("없음"은 추측하지
    말고 그대로 보고). **주의**: TPC:JS가 같은 내용을 다른 비유로 옮긴
    경우가 있다(예: "hemlock 옹이를 옥수수빵 쐐기·호박 망치로 쪼개기"
    비유가 "계란으로 바위 깨기"로 통째 대체됨). 원문이 그 비유를 뒤이어
    직접 풀어서 설명하는 등 비유 자체가 문맥에 필수적이면, 공식본이 있어도
    임시 번역 + `[확인 필요]`를 유지한다 — 어휘 우선순위(참고집 우선)보다
    문맥 일관성이 먼저다.
- **speeches.byu.edu의 PDF 직링크(`/wp-content/uploads/pdf/...`)는 WordPress
  REST API 우회 없이 곧바로 `curl -sL <url> -o file.pdf`로 받아지고, PDF라서
  일반 SPA 페이지의 base64 인코딩 문제도 없다.** `pypdf`로 페이지별 텍스트를
  추출하면 각주까지 포함한 전문을 바로 얻는다. speeches.byu.edu의 일반
  HTML 페이지(REST API 우회가 필요한 경우)와는 별개 케이스이니 URL이
  `.pdf`로 끝나면 API 단계를 건너뛰어도 된다.
- **각주 수십 개짜리 BYU 학술 강연(Joseph Smith Lecture Series 등 Education
  Week 강연)은 총회 말씀과 인용 검증 절차가 다르다.** 이런 글은 조셉 스미스의
  개인 편지·일기·TPJS·동시대인 회고록 등 19세기 1차 사료 인용이 대부분이며,
  이런 인용은 사실상 공식 한국어 번역이 존재하지 않는다. 각 인용마다
  `[확인 필요]`를 붙이면 수십 개가 되어 오히려 신호 대 잡음비가 나빠진다 —
  대신 (1) 성경·교리와 성약처럼 실제 경전인 인용만 공식 한국어로 교체하고
  (2) 나머지 1차 사료 인용은 정확한 직접 번역으로 처리한 뒤 (3) 채팅
  리뷰에 "이 글의 1차 사료 인용은 공식 한국어가 없다"는 사실을 한 번만
  명시한다. 번역 범위(본문만 vs 각주 포함)는 각주가 많으면 먼저
  사용자에게 확인한다.
- **직책 한국어 표기는 비슷한 발음의 타 교단 용어와 혼동하기 쉽다.**
  Presiding Bishop은 "감독회장"이 아니라 "감리 감독"이다("감독회장"은
  한국 감리교 용어). 참고집에 이미 있는 표기가 통념과 다르게 느껴져도
  임의로 통념(다른 교단에서 익숙한 표현)을 따르지 말고, 참고집 표기를
  churchofjesuschrist.org 인물 소개 페이지로 재확인하는 쪽을 우선한다.

## 링크 번역 절차

사용자가 URL을 주면 유형을 판단해 원문을 추출한 뒤 번역한다.
**번역·리뷰·출력 규칙은 영어 원문을 직접 붙여넣었을 때와 동일하다.**
이 절차는 원문 확보 단계만 추가하는 것이다.

```
URL 제공?
├── YouTube (youtube.com / youtu.be)
│   └── scripts/extract-youtube.sh <URL> 실행
│       .cache/<video-id>/meta.json + transcript.txt 읽기
│       → transcript.txt를 원문으로 사용
└── 일반 웹 링크
    └── WebFetch로 페이지 가져오기
        본문(말씀·기사·교과) 추출, 내비·광고·푸터 제거
        → 추출한 본문을 원문으로 사용
```

### YouTube 링크

1. `scripts/extract-youtube.sh <URL>` 실행.
   - 공식 자막 → 자동 자막 → Whisper STT 순으로 영어 스크립트를 만든다.
   - 의존 도구가 없으면 스크립트가 설치 명령을 안내한다.
2. `.cache/<video-id>/meta.json`(title, uploader, description, upload_date)과
   `transcript.txt` 읽기.
3. 메타데이터 보정:
   - **제목**: `meta.title` 기반, 채널명·부제 등 군더더기 정리.
   - **말씀하신 분**: `uploader`는 보통 채널명이므로 신뢰하지 않는다.
     제목·`description`·본문에서 실제 화자를 추론하고, 불확실하면 `[확인 필요]`.
   - **날짜**: `upload_date`는 실제 말씀 날짜와 다를 수 있다.
     `description`·본문 단서로 보정하고, 불확실하면 `[확인 필요]`.
4. 자막·STT가 모두 실패해 스크립트가 비0으로 종료해도 `meta.json`은 남는다.
   이 경우 메타데이터만 사용자에게 보고하고 원문을 직접 받을 수 있는지 묻는다.

### 일반 웹 링크

1. `WebFetch`로 페이지 가져오기.
2. 말씀·기사·교과 본문만 추출. 내비, 사이드바, 푸터, 광고는 제거.
3. 페이지에서 제목·화자·날짜 확인. 불확실하면 `[확인 필요]`.
4. 로그인 필요 또는 접근 불가 시 사용자에게 보고하고 원문을 직접 받는다.

## 번역 결과물 출력 규칙

- **번역 최종본은 `translations/` 폴더에 별도 md 파일로 저장한다.**
  - 파일명: 원문 제목을 기반으로 한 한국어 슬러그. 예: `translations/교회-인공-지능-사용-원리-발표.md`
  - 폴더가 없으면 만든다.
  - 파일에는 번역 본문만 담는다. `[확인 필요]` 표시는 본문 안에 그대로 유지한다.
  - 원문이 함께 필요하면 파일 맨 아래 `---` 구분선 뒤에 "원문" 섹션으로 덧붙여도 된다 (요청 시).
- **번역 본문은 md 파일에만 저장하고 채팅에 다시 옮기지 않는다.**
  사용자는 파일을 직접 열어서 확인한다. 채팅에 번역 본문을 통째로 붙여 넣지 않는다.
- **리뷰 및 확인 사항(검증 완료 항목, `[확인 필요]`로 남긴 항목, 출처 링크,
  통찰 등)만 채팅에 표시한다.**
  - 후속 수정 요청이 오면 파일을 수정하고, 수정 내역과 새 `[확인 필요]` 항목을
    채팅에 간단히 정리한다.
